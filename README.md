## ü§î How to use?

To use the **Unicorn Sparkle** template, you can copy and run the following command in your terminal:

With NPM:

```bash
npm create astro@latest -- --template UXCorpRangel/portfolios-dev/unicorn-sparkle
```

With PNPM:

```bash
pnpm create astro@latest -- --template UXCorpRangel/portfolios-dev/unicorn-sparkle
```

Then enter your portfolio folder, install the dependencies and you're ready! üéâ.

## üõ†Ô∏è Project Structure

- [public/favicons/](./public/favicons/): Contains the favicon files used in the application.

- [public/media/](./public/media/): Stores media files such as videos that are used in the application.

- [public/og/](./public/og/): This folder stores the Open Graph images that are used for previews when the page is shared on social networks.

- [src/components/](./src/components/): Contains reusable components that can be used in different parts of the application.

- [src/data/](./src/data/): This folder stores static data or files containing the information the application needs.

- [src/icons/](./src/icons/): Includes the icons used in the application.

- [src/images/](./src/images/): Contains all the images used in the application, whether for backgrounds, graphics, or other visual elements.

- [src/layouts/](./src/layouts/): Stores the general layout components of the application, such as complete page structures to maintain visual consistency across different pages.

- [src/locales/](./src/locales/): Folder dedicated to localization or internationalization files.

- [src/pages/](./src/pages/): Contains the main sections of each page of the application. For example, [_home](./src/pages/_home/) contains the sections of the home page.

- [src/routes/](./src/routes/): This folder manages the application's routes.

- [src/scripts/](./src/scripts/): Contains additional scripts necessary for the project's functionality. These scripts may include utilities, helper functions, or any logic not directly related to the UI.

- [src/styles/](./src/styles/): Stores the style files that define the visual appearance of the application. Here colors, fonts, margins, and other visual elements are defined.

## üì¶ Dependencies

Discover the main Astro integrations used in this project:

- [@astrolicious/i18n](https://www.npmjs.com/package/@astrolicious/i18n): It is an internationalization solution for Astro projects. It facilitates the creation of multilingual sites, allowing management of multiple languages and content translation.

- [@playform/compress](https://www.npmjs.com/package/@playform/compress): It is a tool that focuses on compressing assets, such as images, scripts and styles, during the build process.

- [astro-compressor](https://www.npmjs.com/package/astro-compressor): It is an integration that handles compressing files to Brotli and Gzip during the build process.

- [astro-icon](https://www.npmjs.com/package/astro-icon): It is an integration that facilitates the management and insertion of SVG icons in your Astro project.

- [astro-seo-schema](https://www.npmjs.com/package/astro-seo-schema): is an integration that automatically generates structured schema data ([schema.org](https://schema.org/)) for your Astro website.

- [astro-sitemap](https://www.npmjs.com/package/astro-sitemap): Automatically generates an XML sitemap for your Astro website.

- [astro-tunnel](https://www.npmjs.com/package/astro-tunnel): Allows exposing your local Astro project through a public URL.

- [sanitize.css](https://www.npmjs.com/package/sanitize.css): It is a stylesheet that offers a reset and normalization of default CSS styles in browsers. It ensures a more consistent appearance across different browsers, eliminating differences in default styles, allowing for more predictable and coherent UI development.
